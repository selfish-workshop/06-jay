# 페르소나

## 페르소나 1 — 용사 "김대리"

**기본 정보**
- 이름: 김민지 (28)
- 직책: 마케팅 대리 (3년차)
- 회사: 50인 규모 B2C 스타트업
- 보고 대상: 마케팅 팀장

**하루 루틴**
- 09:00 출근 → 커피 → 슬랙 확인
- 09:30 스탠드업 미팅 (구두 보고)
- 10:00 업무 시작
- 18:30 퇴근 시도... 보고서 남아있으면 좌절

**프로덕트 관련 행동**
- 매일 쓰는 툴: Slack, Notion, Figma, Google Sheet
- 새 툴 도입에 개방적이지만 3일 안에 가치 없으면 버림
- 모바일보다 데스크탑 중심

**페인 포인트**
1. "오늘 뭐 할지" 정리하는 것 자체가 피곤
2. 슬랙 보고서 쓰는 시간이 업무 시간의 10% 차지
3. 팀장이 계속 물어봐서 답변하느라 집중 깨짐
4. 중간에 막히면 어떻게 공유할지 애매 (너무 작으면 과한 보고, 너무 크면 놓침)

**목표**
- 보고 시간 최소화
- 오늘 뭘 해야 할지 시각적으로 정리
- 막힐 때 팀장에게 자연스럽게 신호 보내기

**"퇴근을 향하여"가 해결하는 것**
- 업무 입력이 "캐릭터 만들기" 느낌이라 거부감 낮음
- 퀘스트 카드로 할 일 한눈에
- 막힘 상태 토글 하나로 팀장에게 신호 완료

**거부감 포인트**
- "게임처럼 포장된 감시 툴" 같으면 즉시 이탈
- 입력 필드가 5개 넘어가면 귀찮음
- 강제 알림이나 리마인더는 싫어함

## 페르소나 2 — 보스 "박팀장"

**기본 정보**
- 이름: 박준호 (38)
- 직책: 마케팅 팀장 (12년차)
- 팀 규모: 6명
- 보고 대상: CMO, 대표

**하루 루틴**
- 08:50 출근 → 전체 메일 확인
- 09:30 팀 스탠드업 (팀원들 한마디씩)
- 10:00 개인 업무 + 팀원 Q&A
- 14:00 상사 보고 / 타부서 미팅
- 18:00 팀원 퇴근 확인

**프로덕트 관련 행동**
- 정보 밀도 높은 대시보드 좋아함
- 일일이 물어보기 귀찮지만 상황은 알고 싶음
- 팀원 관리 툴 여러 개 써본 경험 (Asana, Jira, ClickUp)

**페인 포인트**
1. 팀원에게 "어떻게 돼가?" 묻는 게 서로 피곤
2. 주간 보고 때문에 금요일 오후 다 날아감
3. 누가 막혔는지 늦게 알게 되면 대응 타이밍 놓침
4. 팀원 감정노동 유발하지 않으면서도 상황 파악하고 싶음

**목표**
- 팀원 상황을 "방해하지 않고" 파악
- 막힌 팀원을 빠르게 발견해 지원
- 상사 보고용 데이터 자동 확보

**"퇴근을 향하여"가 해결하는 것**
- 보스 대시보드 한 화면으로 팀 전체 스캔
- 막힘 뱃지로 즉시 지원 타겟 식별
- 팀원이 보고서 안 써도 됨 → 팀원 심리적 부담 감소

**거부감 포인트**
- 팀원이 "감시당한다"고 느끼면 안 됨
- 대시보드가 복잡하면 안 봄
- 매일 로그인 강제하면 부담
