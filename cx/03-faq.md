# FAQ — 퇴근을 향하여

발표 및 체험 시 예상되는 질문과 답변.

---

## 기능 관련

**Q1. 이거 진짜 게임이에요?**
아니요. 업무관리 도구입니다. 업무 입력의 부담을 낮추기 위해 게임 연출을 입혔을 뿐, 핵심은 "업무 진행 상황 관리"입니다.

**Q2. 캐릭터를 왜 골라요? 꾸미는 건가요?**
꾸미기 아닙니다. "캐릭터를 고른다"고 말하지만 실제로는 **오늘의 업무를 입력하는 단계**입니다. 게임 연출로 포장한 것뿐이에요.

**Q3. 팀원 데이터는 진짜인가요?**
데모 버전에서는 **더미 데이터**입니다. 보스모드 대시보드의 팀원 4~5명은 미리 세팅된 가상 인물이고, 본인(용사모드) 데이터만 실시간 반영됩니다.

**Q4. 데이터는 어디에 저장되나요?**
브라우저 **localStorage**에만 저장됩니다. 서버 없음. 다른 브라우저나 디바이스에서는 데이터를 볼 수 없고, 브라우저 캐시 지우면 날아갑니다.

**Q5. 모바일에서도 되나요?**
데모 버전은 **데스크탑 전용**입니다. 최소 1024px 해상도 필요. 모바일 최적화는 의도적으로 뺐습니다.

## 컨셉 관련

**Q6. 이거 감시 툴 아니에요?**
감시 툴이 **아니게 만드는 것**이 핵심 설계 원칙입니다:
- 용사모드는 직원이 주도적으로 입력
- 보스모드는 읽기 전용 (팀원 데이터 수정 불가)
- 실시간 감시 알림 없음 (팀원이 로그인 안 하면 리더도 못 봄)
→ 직원이 "자발적으로 공유하는" 구조를 유지합니다.

**Q7. 팀장이 내 모든 걸 볼 수 있다는 거잖아요?**
보스모드에서 보이는 건 **본인이 입력한 퀘스트명, 진행률, 막힘 여부, 보고 메모**뿐입니다. 사적인 내용 입력을 강제하지 않고, 퀘스트 이름은 본인이 자유롭게 씁니다.

**Q8. 왜 RPG 테마예요?**
업무 보고는 **피곤함의 대명사**입니다. RPG 퀘스트 구조는:
1. 할 일(퀘스트)과 난이도가 명확하고
2. 진행 상태가 시각적으로 드러나며
3. 완료 시 소소한 성취감이 있고
4. "게임" 문맥이 심리적 부담을 낮춥니다.
업무 관리의 구조와 RPG의 구조가 겹친다는 발견이 이 서비스의 출발점입니다.

## 데모/발표 관련

**Q9. 발표에서 뭘 봐야 하나요?**
5분 데모에서 주목할 포인트:
1. **모드 대비** — 용사/보스 첫 화면
2. **캐릭터=업무 전환** — S2의 반전
3. **오프닝 시퀀스** — 첫인상
4. **모드 간 데이터 연동** — 용사모드 입력이 보스모드에 반영되는 순간

**Q10. 이걸로 실제 사업 할 건가요?**
현재는 **워크숍 학습 목적의 데모**입니다. 다만 컨셉이 검증되면 SaaS로 확장 가능성을 열어뒀습니다. 수익 모델은 `strategy/` 폴더 참조.

## 문제 해결

**Q11. 오프닝이 너무 길어요.**
우상단 **Skip** 버튼을 누르거나 **ESC**/**Space** 키를 누르면 즉시 건너뜁니다. 재방문 시에는 "다음부터 자동 건너뛰기"를 체크할 수 있습니다.

**Q12. 데모 다시 처음부터 보고 싶어요.**
용사모드 우상단 ⚙ 설정 → "오늘의 퀘스트 초기화"를 누르면 localStorage가 지워지고 첫 화면부터 다시 시작합니다.

**Q13. 보스모드에 내 데이터가 안 보여요.**
- 먼저 용사모드에서 퀘스트를 입력했는지 확인
- 같은 브라우저·같은 창에서 모드 변경을 했는지 확인
- 브라우저 시크릿/프라이빗 모드면 localStorage가 모드 변경 시 초기화될 수 있음

**Q14. 퀘스트가 안 저장돼요.**
브라우저 설정에서 **쿠키/사이트 데이터 차단**이 켜져 있으면 localStorage가 동작하지 않습니다. 설정을 확인해주세요.
