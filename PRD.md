# 퇴근을 향하여 — PRD

## 1. 서비스 개요

**퇴근을 향하여**는 업무 보고의 부담을 줄이고 관리자의 가시성을 높이기 위한 게임형 업무관리 웹서비스 데모다. 직원은 게임을 플레이하는 느낌으로 업무를 입력·관리하고, 팀장·대표는 보스 시점에서 팀원의 퀘스트 수행 현황을 확인한다. 단순히 웃긴 컨셉이 아니라, **입력해야 할 업무 정보를 더 자연스럽게 구조화하는 UX 실험**이 목적이다.

## 2. 폴더 구조

이 레포는 5개 팀(파트)에 따른 5개 영역으로 나뉜다.

- `planning/` — 기획 (무엇을 만들 것인가)
- `design/` — 디자인 (어떻게 보여줄 것인가)
- `cx/` — CX (누가 어떻게 사용할 것인가)
- `marketing/` — 마케팅 (어떻게 알릴 것인가)
- `strategy/` — 전략 (어떻게 지속 가능하게 만들 것인가)

각 영역의 상세는 해당 폴더에서 작성된다.

## 3. 기획 (Planning)

직원은 용사모드에서 오늘의 업무를 캐릭터 생성 단계로 위장해 입력하고, 퀘스트 화면에서 진행률·챌린지·난이도를 관리한다. 팀장은 보스모드에서 팀원별 퀘스트 현황을 대시보드로 확인한다. MVP 범위는 **모드 선택 → 캐릭터 생성(업무 입력) → 오프닝 시퀀스 → 퀘스트 화면 / 보스 대시보드**까지다.

상세는 `planning/` 폴더 참조.

## 4. 디자인 (Design)

픽셀 RPG 톤에 모던 웹 UX를 섞은 하이브리드 비주얼. 직원에게는 게임풍으로 친근하게, 관리자에게는 정보 밀도 높은 대시보드로 보여야 한다. 화면 흐름은 모드 선택 → 캐릭터 생성 → 오프닝 → 본 화면의 단방향 구조를 기본으로 한다.

상세는 `design/` 폴더 참조. [작성 예정]

## 5. CX (Customer Experience)

타겟 페르소나는 "보고서 쓰기 싫은 직원"과 "팀원 상황을 계속 묻기 미안한 리더" 두 축이다. 사용자 여정의 핵심은 **부담 없이 업무를 입력 → 퀘스트 수행 → 자동으로 리더에게 보여지는** 흐름이다.

상세는 `cx/` 폴더 참조. [작성 예정]

## 6. 마케팅 (Marketing)

첫 사용자는 소규모 팀(5~15인)과 스타트업 대표를 타겟으로 SNS·커뮤니티 데모 영상으로 확보한다. "퇴근이 1시간 빨라진다"는 핵심 카피로 공감형 바이럴을 노린다.

상세는 `marketing/` 폴더 참조. [작성 예정]

## 7. 전략 (Strategy)

데모 단계에서는 수익화보다 피드백 확보가 목적이다. 장기적으로는 SaaS 구독(팀당 월정액) 또는 기업 내부 툴 라이선스 판매를 고려한다. 실적관리 연동까지 확장 가능성을 남겨둔다.

상세는 `strategy/` 폴더 참조. [작성 예정]

## 8. 역할 배정 (가상)

| 팀 | 담당 영역 | 작업 폴더 |
|---|---|---|
| PM (본인) | 전체 기획·의사결정 | 루트 |
| 기획 팀 | 기능 정의, MVP 범위 | `planning/` |
| 디자인 팀 | UI/UX, 비주얼 | `design/` |
| CX 팀 | 페르소나, 여정맵, FAQ | `cx/` |
| 마케팅 팀 | 런칭, 채널 | `marketing/` |
| 전략 팀 | 수익 모델, 성장 | `strategy/` |
