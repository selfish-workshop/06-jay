# 핵심 인터랙션

## 원칙

1. **피드백 즉시** — 클릭/입력 반응은 100ms 이내
2. **게임풍 장치는 포인트로** — 과하면 업무 툴 신뢰도 떨어짐
3. **Skip 가능해야 함** — 반복 시연 시 스트레스 금지
4. **키보드만으로도 데모 가능** — 발표 현장 마우스 실수 방지

---

## 주요 인터랙션 목록

### I1. 모드 카드 선택
- Hover: 카드 1.03배 확대 + 액센트 컬러 외곽 글로우
- Click: 0.95배 눌림 → 원크기 복귀 → 페이드 전환

### I2. 퀘스트 입력 — 추가/삭제
- "+ 퀘스트 추가" 클릭 → 새 입력 카드 아래 슬라이드인
- 기존 카드 우상단 ✕ → 카드가 좌측으로 슬라이드 아웃
- 입력 중 탭 키로 필드 이동 가능 (접근성)

### I3. "소환하기" 버튼
- 최소 1개 퀘스트 입력 전: 회색, 비활성
- 1개 이상 입력 후: 옐로 액센트 + 살짝 펄스 애니메이션 (2초 주기)
- 클릭 시: 화면 페이드 아웃 → S3 오프닝 시작

### I4. 오프닝 Skip
- 키보드 ESC 또는 Space로도 Skip 가능 (발표 편의)
- Skip 클릭 시: 즉시 검은 배경 페이드 아웃 → S4 진입
- 두 번째 방문자: Skip 버튼 옆에 "다음부터 자동 건너뛰기" 체크박스

### I5. 퀘스트 카드 — 상태 변화
- 진행률 바 업데이트: 0.4초 ease-out 애니메이션
- 100% 도달 순간: 카드 전체 그린 글로우 + "✓ 클리어" 배지 팝업
- 막힘 상태: 카드 테두리 빨강 + 아이콘 흔들림(shake) 1회

### I6. 퀘스트 상세 모달
- 오픈: 배경 blur + 모달 아래서 위로 슬라이드인
- 진행률 슬라이더: 5단계 snap (0/25/50/75/100)
- 저장 버튼 누르면 모달 즉시 닫히고 카드 업데이트 애니메이션

### I7. 모드 변경 버튼
- 우상단 고정
- 클릭 시: 현재 화면 페이드 → S1 모드 선택 화면
- 데이터는 유지 (localStorage)

### I8. 보스 대시보드 — 팀원 카드
- 정렬 애니메이션: 막힘 팀원이 상단으로 밀려 올라옴 (0.5초)
- Hover: 카드 살짝 뜸 (shadow)
- 막힘 뱃지: 붉은 점 1초 주기 pulse

### I9. 데모 초기화 (⚙ 설정)
- 설정 드롭다운 → "오늘의 퀘스트 초기화"
- 확인 다이얼로그: "정말 초기화할까요? (발표 재시연용)"
- 확인 시 localStorage 클리어 → S1로 복귀

## 마이크로 디테일 (여유 되면)

- 퀘스트 카드 생성 시 "✨" 파티클 이펙트
- 보스 대시보드 진입 시 카드들이 스태거(순차) 페이드인
- 오프닝 시퀀스 배경에 희미한 픽셀 별 깜빡임
- 완료 퀘스트 클리어 시 효과음 (8-bit 짧은 jingle)
  - **주의**: 발표장에서 음소거 필수 → 기본 off, 설정에서 on

## 접근성 체크

- 모든 인터랙티브 요소: 키보드 포커스 가능 (Tab)
- 색상에만 의존 X: 막힘 = 빨강 + 아이콘 + 텍스트 "막힘"
- 애니메이션: `prefers-reduced-motion` 감지 시 축소
