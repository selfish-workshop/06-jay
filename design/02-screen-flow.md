# 화면 흐름

## 전체 흐름도

```
[진입]
   ↓
[S1. 모드 선택]
   ├─→ 용사모드 ─→ [S2. 캐릭터 생성(업무 입력)] ─→ [S3. 오프닝] ─→ [S4. 퀘스트 메인] ⇄ [S4-1. 퀘스트 상세 모달]
   └─→ 보스모드 ─→ [S5. 보스 대시보드] ⇄ [S5-1. 팀원 상세 모달]

모든 화면 우상단 "모드 변경" 버튼 → [S1]로 복귀
```

## 화면별 목적과 체류 시간

| ID | 화면 | 목적 | 평균 체류 |
|---|---|---|---|
| S1 | 모드 선택 | 서비스 구조 즉시 이해 | 2초 |
| S2 | 캐릭터 생성 = 업무 입력 | 부담 없이 오늘 업무 입력 | 40~60초 |
| S3 | 오프닝 시퀀스 | 세계관 몰입 | 8초 (또는 Skip) |
| S4 | 퀘스트 메인 | 업무 진행 관리 (메인 체류 화면) | 제한 없음 |
| S4-1 | 퀘스트 상세 모달 | 진행률·막힘·메모 업데이트 | 10~20초 |
| S5 | 보스 대시보드 | 팀 전체 한눈에 파악 | 30~60초 |
| S5-1 | 팀원 상세 모달 | 특정 팀원 퀘스트 확인 | 15~30초 |

## 전환 원칙

- **S1 → S2**: 슬라이드인 (좌에서 우)
- **S2 → S3**: 페이드 아웃 → 검은 배경
- **S3 → S4**: 텍스트 올라가다 페이드 → 퀘스트 화면 페이드인
- **S4 ⇄ S4-1**: 모달 (backdrop blur)
- **모드 변경 버튼**: 어느 화면에서든 → 페이드 → S1

## 데모 시나리오 경로 (발표 5분)

```
S1(용사) → S2(업무 3개 입력) → S3(Skip) → S4(카드 보기)
  → S4-1(진행률 조정) → 모드 변경 버튼
S1(보스) → S5(본인 데이터 반영 확인) → S5-1(막힘 팀원 확인)
```

**총 클릭 수**: 약 8회
**필수 가시 요소**: 모드 대비, 퀘스트 카드, 진행률 변화, 보스 대시보드 빨간 뱃지

## 반응형

- **우선**: 1440×900 (발표 노트북)
- **최소**: 1024px
- **모바일**: 의도적 제외 (데모 범위 밖)
