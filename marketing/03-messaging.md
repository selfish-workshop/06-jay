# 런칭 메시지와 카피

## 핵심 메시지 피라미드

```
        [한 줄 태그라인]
       퇴근이 1시간 빨라진다
       ─────────────────
      [세 가지 핵심 약속]
   - 업무 보고 부담 제거
   - 팀장 가시성 확보
   - 재미있는 UX로 지속
   ───────────────────────
        [증거]
   - 데모 5분이면 체감
   - 첫 방문 2분 안에 가치 전달
   - 감시 툴 아닌 자발적 공유 구조
```

---

## 상황별 카피

### 랜딩 페이지 히어로
**헤드라인**: 오늘의 용사를 소환하세요
**서브**: 업무를 퀘스트처럼 관리하고, 팀장은 보스 시점에서 확인합니다
**CTA 버튼**: 체험해보기

### 30초 광고 훅 (5가지 버전)
1. "업무 보고 쓰는 시간이 아까우신가요?"
2. "팀원이 지금 뭐 하는지 궁금한데, 묻기는 미안하시죠?"
3. "업무 관리를 게임으로 바꿔봤습니다."
4. "퇴근을 향한 퀘스트, 시작하시겠어요?"
5. "아침마다 보고서 쓰는 거, 지겨우시죠?"

### SEO 메타 디스크립션
"업무를 RPG 퀘스트처럼 관리하고, 팀장은 보스 시점에서 팀원의 진행 상황을 확인하는 게임형 업무관리 서비스. 보고 부담 없이, 감시 없이."

## 페르소나별 메시지

### 용사 (직원)에게
- "보고서 안 써도 팀장님이 내 상황을 알게 됩니다"
- "업무 입력이 이렇게 가벼울 수 있습니다"
- "막힌 일도 조용히 신호 보내는 법"

### 보스 (리더)에게
- "묻지 않고도 팀 상황을 파악하세요"
- "팀원이 감시당한다고 느끼지 않는 유일한 대시보드"
- "막힌 팀원을 놓치지 마세요"

### 조직 의사결정자에게
- "주간 보고 회의, 정말 필요하신가요?"
- "업무 가시성과 심리적 안전감을 동시에"
- "도입 5분이면 끝나는 가벼운 실험"

## 런칭 타임라인

### 출시 전 2주
- **D-14**: 티저 영상 제작 시작
- **D-10**: 랜딩 페이지 오픈 (대기자 수집)
- **D-7**: 첫 인플루언서 3명에게 프리뷰 접근 제공
- **D-5**: 디스카이엇 메이커 로그 시작
- **D-3**: 트위터/X 카운트다운 시작

### 출시 당일 (D-Day)
- **09:00**: 트위터 런칭 스레드 포스팅
- **10:00**: 링크드인 런칭 글
- **12:00**: 인스타그램 릴스 업로드
- **14:00**: 디스카이엇 런칭 공지
- **18:00**: 유튜브 쇼츠 업로드

### 출시 후 1주
- **D+1**: 첫 사용자 피드백 수집
- **D+3**: 주요 피드백 반영 업데이트
- **D+5**: 2차 콘텐츠 배포
- **D+7**: 런칭 1주 리포트 공유 (디스카이엇)

## 리스크 메시지 대응

### "이거 감시 툴 아니에요?"
→ "팀원 주도 입력 구조입니다. 리더는 팀원이 공유한 것만 봅니다."

### "회사에서 강제로 쓰게 할까 봐 걱정돼요"
→ "의무 도입은 설계 의도에 반합니다. 팀원이 자발적으로 쓸 때 가장 효과적이에요."

### "데이터 어디 저장돼요?"
→ "현재 데모는 브라우저에만 저장합니다. 서버 없음."

### "진짜 게임이에요?"
→ "업무 툴입니다. 게임 UX는 입력 부담을 낮추는 장치일 뿐이에요."
