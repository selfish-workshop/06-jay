# MVP 범위

발표 데모에서 **반드시 동작**해야 하는 범위 vs 추후로 미룰 것을 명확히 분리.

---

## ✅ MVP 필수 (Must Have)

데모에서 이게 없으면 컨셉 전달이 불가능한 항목.

### 화면
1. **모드 선택 화면** — 용사/보스 2개 카드
2. **캐릭터 생성 (= 업무 입력) 화면** — 퀘스트 입력 폼 (최소 필드만: 퀘스트명, 난이도, 타입)
3. **오프닝 시퀀스** — Skip 버튼 포함, 5~8초 크레디트 애니메이션
4. **퀘스트 메인 화면 (용사모드)** — 카드 리스트 + 진행률 업데이트 모달
5. **보스 대시보드** — 팀원 카드 그리드 + 막힘 뱃지 + 상세 모달

### 기능
- 모드 간 이동 (우상단 "모드 변경" 버튼)
- 오프닝 Skip 버튼 (항상 클릭 가능)
- 퀘스트 카드 진행률 업데이트 (25% 단위)
- 막힘 상태 토글 + 메모 한 줄
- 용사모드에서 입력한 데이터가 보스모드 본인 카드에 반영 (localStorage)
- 더미 팀원 4~5명 (하드코딩)

### 데이터
- **localStorage만 사용** — 서버/DB 없음
- 세션 간 유지 불필요 (데모 중 초기화 버튼으로 리셋 가능)

---

## 🟡 있으면 좋은 것 (Should Have)

시간 남으면 추가, 없어도 데모는 가능.

- 오프닝 시퀀스 자동 Skip 옵션 (재방문자용)
- 퀘스트 클리어 이펙트 (애니메이션)
- 퀘스트 카운터 상단 요약 ("완료 3 / 진행 2 / 막힘 1")
- 제한 시간 D-Day 표시
- 모바일 반응형 레이아웃

---

## 🔴 추후 (Won't Have — This Release)

컨셉상 매력적이지만 데모 밖으로 밀어냄.

- 로그인/회원가입
- 실제 팀원 초대
- 서버 API / DB 연동
- 실적관리 대시보드 (전략 팀에서 장기 방향으로만 언급)
- 경험치/레벨업 시스템
- 다중 세션 동기화 (실시간 푸시)
- 알림 기능
- 다국어 지원
- 다크모드
- 관리자 권한 체크 (용사가 보스모드 들어가도 OK)

---

## 구현 기술 스택 (권장)

- **Frontend**: Vanilla HTML + CSS + JS 또는 React (Vite)
  - 발표 직전 배포 편의성 고려 → 단일 HTML 파일로 끝낼 수도 있음
- **상태 관리**: localStorage 직접 사용 (JSON 직렬화)
- **스타일**: 픽셀 RPG 느낌 → Google Fonts `Press Start 2P` + 커스텀 CSS
- **배포**: GitHub Pages 또는 Vercel
- **빌드 시간**: 프로토타입 기준 2~4시간
