# 기능 명세서 — 퇴근을 향하여

각 기능의 동작을 정의한다. MVP 범위는 `03-mvp-scope.md` 참조.

---

## F1. 모드 선택 화면 (Entry)

**목적**: 서비스 진입 즉시 "직원용 / 관리자용"으로 나뉜다는 것을 직관적으로 전달.

**동작**:
- 첫 진입 시 전체 화면에 2개의 큰 카드 표시
  - 좌측: **용사모드 (Member)** — 검/방패 아이콘
  - 우측: **보스모드 (Leader)** — 왕관/성 아이콘
- 카드 클릭 → 해당 모드 플로우로 진입
- 뒤로가기 시 언제든 이 화면으로 복귀 가능 (우상단 "모드 변경" 버튼)
- 실제 권한 체크 없음 (데모 목적)

**엣지 케이스**:
- 직원이 보스모드를 눌러볼 수 있음 → 데모 컨셉상 허용
- localStorage에 마지막 선택 모드 저장 → 재방문 시 빠른 진입 옵션 제공

## F2. 캐릭터 생성 (= 오늘의 업무 입력)

**목적**: "캐릭터 만든다"는 게임 경험으로 업무 입력 부담을 낮춘다.

**동작**:
- 용사모드 선택 후 바로 진입
- 화면 구성:
  - 상단: "오늘의 용사를 소환하세요" 카피
  - 중앙: 업무 입력 폼 (아래 필드)
  - 하단: "소환하기" 버튼
- 입력 필드:
  - **퀘스트명** (필수) → 업무명. 예: "클라이언트 제안서 작성"
  - **난이도** (필수) → 1~5 별점 또는 쉬움/보통/어려움/보스급
  - **퀘스트 타입** (필수) → 메인 / 서브 / 일일반복 / 보스전
  - **제한 시간** (선택) → 마감 일정
  - **챌린지** (선택) → 리스크/막힘 요소 한 줄
- 퀘스트는 **여러 개 추가 가능** (+ 버튼으로 입력 행 추가)
- 최소 1개 이상 입력해야 "소환하기" 버튼 활성화

**엣지 케이스**:
- 0개 입력 상태 → "빈 손으로는 전장에 갈 수 없어요" 토스트
- 10개 초과 입력 시도 → "오늘 하루치는 충분합니다" 경고 (실적 관점 무리 방지)

## F3. 오프닝 시퀀스

**목적**: 업무를 퀘스트 세계로 변환하는 세계관 몰입.

**동작**:
- 캐릭터 생성 완료 직후 자동 재생
- 영화 크레디트 스타일: 검은 배경 + 흰 텍스트가 아래에서 위로 스크롤
- 내용 예시 (3~5줄 정도, 약 8초):
  - "202X년, 사무실이라는 던전에서"
  - "용사들은 오늘도 퀘스트를 받았다"
  - "그들의 진짜 목표는 단 하나 —"
  - "퇴근"
- **Skip 버튼**: 우상단 고정, 항상 클릭 가능. 클릭 시 즉시 본 화면으로.
- 끝까지 본 경우에도 자동으로 본 화면 전환
- localStorage에 "한 번 본 사용자" 플래그 → 다음 방문 시 Skip 버튼 대신 "자동 Skip" 옵션 노출

**엣지 케이스**:
- 뒤로가기로 돌아오면 캐릭터 생성 화면
- 재생 중 창 크기 변경 → 텍스트 위치 자동 재정렬

---

## F4. 퀘스트 메인 화면 (용사모드 본 화면)

**목적**: 입력한 업무를 퀘스트 카드로 시각화하고 진행 상태를 업데이트.

**동작**:
- 화면 구성:
  - 상단: 사용자 정보 (닉네임, 오늘의 총 퀘스트 수, 완료/진행중/대기)
  - 중앙: 퀘스트 카드 리스트
  - 하단: "퀘스트 추가" 버튼
- 퀘스트 카드 요소:
  - 타입 배지 (메인/서브/일일/보스)
  - 퀘스트명
  - 난이도 표시
  - 진행률 바 (0~100%)
  - 챌린지 아이콘 (있을 경우)
  - 제한 시간 D-Day
- 카드 클릭 → 상세 모달
  - 진행률 조정 슬라이더 (0% / 25% / 50% / 75% / 100%)
  - 막힘 여부 체크박스 ("함정에 빠졌다")
  - 보고 메모 (한 줄 입력)
  - 저장 버튼
- 100% 완료 시 카드에 "클리어!" 이펙트 + 상단 카운터 업데이트

**엣지 케이스**:
- 모든 퀘스트 완료 시 → "오늘의 퇴근 가능" 축하 화면 토스트
- 진행률 0%에서 100%로 한 번에 점프 → 허용 (데모 편의)

## F5. 보스 대시보드 (보스모드 본 화면)

**목적**: 팀원의 퀘스트 현황을 한눈에 파악.

**동작**:
- 화면 구성:
  - 상단: 팀 전체 진행도 (총 퀘스트 N개 / 완료 N개 / 막힘 N개)
  - 중앙: 팀원 카드 그리드 (3x2 or 반응형)
  - 팀원 카드 요소:
    - 이름
    - 오늘의 주요 퀘스트 (메인 퀘스트 제목)
    - 진행률 바
    - 막힘 알림 뱃지 (있을 경우 빨강)
  - 카드 클릭 → 해당 팀원의 전체 퀘스트 상세 목록 모달
- **데모용 팀원 데이터는 고정 4~6명** (localStorage 또는 하드코딩)
  - 본인(로그인한 용사) + 더미 팀원 3~5명

**엣지 케이스**:
- 용사모드에서 입력한 퀘스트가 보스모드 "본인" 카드에 반영되어야 함 (localStorage 공유)
- 더미 팀원은 랜덤 진행률로 미리 세팅

---

## F6. 전역 UX 요소

- **모드 전환**: 우상단 "모드 변경" 버튼 → 모드 선택 화면 복귀
- **Skip 버튼**: 오프닝 시퀀스에서만 노출
- **리셋**: 설정에 "오늘의 퀘스트 초기화" (데모 재시연용)
- **반응형**: 최소 1024px 데스크탑 우선, 모바일은 차선
