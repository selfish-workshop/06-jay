# 사용자 시나리오

주요 사용자 2명의 5W1H 기반 시나리오. 데모 발표 흐름과 직결된다.

---

## 시나리오 1 — 용사 (직원): 김대리

**Who**: 3년차 마케팅 대리. 매일 아침 슬랙에 업무 보고 쓰는 게 제일 귀찮음.
**When**: 월요일 오전 9시 15분. 출근 후 커피 한 잔.
**Where**: 사무실 노트북.
**Why**: 오늘 할 일을 정리해야 하고, 팀장이 또 "오늘 뭐 하냐" 물어볼 게 뻔함.
**How**:

1. 브라우저에서 "퇴근을 향하여" 북마크 클릭
2. 모드 선택 화면 → **용사모드** 카드 클릭
3. 캐릭터 생성 화면에서 오늘의 퀘스트 3개 입력:
   - "상반기 캠페인 기획안 초안" / 보스전 / 제한시간 D-2 / 챌린지: 대표님 리뷰 대기
   - "인스타 광고 소재 2안 제작" / 메인 / D-Day
   - "경쟁사 리서치 정리" / 서브 / D-3
4. "소환하기" 클릭 → 오프닝 시퀀스 재생 → Skip (두 번째 방문이라 자동 스킵 체크)
5. 퀘스트 메인 화면 진입. 카드 3장 확인.
6. 오전 11시, 인스타 소재 1안 완성 → 해당 카드 클릭 → 진행률 50%로 조정 → 저장
7. 오후 2시, 기획안 초안에서 막힘 → 카드 클릭 → "함정에 빠졌다" 체크, 메모: "대표님 피드백 대기"
8. 오후 5시, 모든 퀘스트 완료 → 클리어 이펙트 → "퇴근 가능" 축하 토스트

**기대 효과**: 슬랙 보고 안 썼는데도 팀장이 내 상황을 알고 있음. 보고 안 해도 된다는 심리적 해방.

## 시나리오 2 — 보스 (팀장): 박팀장

**Who**: 마케팅팀장. 팀원 6명. 매일 누가 뭐 하는지 궁금하지만 일일이 물어보기 미안함.
**When**: 같은 날 오전 10시. 스탠딩 미팅 직후.
**Where**: 본인 자리.
**Why**: 팀원들이 오늘 뭐 집중하고 있는지 + 막힌 팀원 있는지 빠르게 확인하고 싶음.
**How**:

1. "퇴근을 향하여" 열기 → 모드 선택 → **보스모드** 선택
2. 대시보드 진입. 상단 요약: "팀 전체 퀘스트 18개 / 완료 4 / 진행중 12 / 막힘 2"
3. 팀원 카드 그리드에서 **빨간 막힘 뱃지**가 붙은 카드 2개 바로 눈에 들어옴
4. 김대리 카드 클릭 → 상세 모달 → "대표님 피드백 대기"라는 메모 확인
5. → 대표님한테 직접 상황 전달 (김대리 대신 병목 제거)
6. 나머지 팀원들은 정상 진행 중 확인 → 대시보드 닫음

**기대 효과**: 팀원에게 "어디까지 했냐" 묻지 않고도 전체 파악. 리더의 감시자 느낌 없이 지원자 역할.

## 시나리오 3 — 발표 데모 진행 (워크숍 발표자)

**Who**: 워크숍 발표자 본인.
**When**: 발표 시간 5분.
**Where**: 발표 화면.
**How**:

1. 모드 선택 → 용사모드 (10초 임팩트)
2. 미리 준비된 샘플 업무 3개 빠르게 입력 (30초)
3. 오프닝 시퀀스 풀 재생 (8초, 첫인상 장치)
4. 퀘스트 화면에서 1개 진행률 조정 시연 (20초)
5. 모드 변경 → 보스모드 (본인 카드에 방금 업데이트한 내용 반영됨을 보여줌)
6. 막힘 시나리오 더미 데이터로 "이렇게 보인다" 시연 (30초)
7. 마무리 메시지: "업무 보고가 힘들다는 현실 문제를 게임 UX로 가볍게 바꿨다"

**기대 효과**: 5분 안에 컨셉 + 실제 동작 + 강점 전달 완료.
